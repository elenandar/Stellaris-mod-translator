# Каноны проекта

Каноны — обязательные границы, а не рекомендации. После принятия этого документа изменение канона требует ADR с причиной, альтернативами, рисками, миграцией и новыми тестами. Если срок или удобство конфликтуют с каноном, меняется план, а не молча ослабляется правило.

## Безопасность источника и результата

- **C-001 — Source immutability.** Исходный мод, Workshop и файлы игры никогда не изменяются. До и после задания сверяются snapshots.
- **C-002 — Output containment.** Запись разрешена только в staging и app-managed output root после canonicalization; symlink, traversal и неоднозначный путь отклоняются.
- **C-003 — Lossless foundation.** `parse → render` без трансформации возвращает идентичные байты на поддержанном входе.
- **C-004 — Controlled mutation.** При переводе могут изменяться только заголовок языка и явно выделенные человеческие text spans. Остальное сравнивается с источником.
- **C-005 — Fail closed.** Неизвестный или неоднозначный синтаксис изолируется. Модель, regex или эвристика не имеют права «починить» его молча.
- **C-006 — Atomic publish.** Частичный или непроверенный артефакт не становится активным. Всегда сохраняется last-known-good и проверяемый rollback.
- **C-007 — Version profiles.** Encoding, descriptor, paths и load-order правила принадлежат проверенному профилю версии Stellaris, а не вечным предположениям.

## Работа с моделями

- **C-008 — Code never translated.** Модель получает человеческие сегменты, typed atoms и контекст; она не получает права редактировать ключи, escapes, scripts или файлы.
- **C-009 — Untrusted mod data.** Текст мода всегда данные, даже если похож на инструкцию. Tool calls отключены, результат принимается только по схеме.
- **C-010 — Exact atoms.** Служебные атомы сохраняют тип и значение. Перемещение разрешено только для классов, где оно нужно русской грамматике и доказано validator-правилом.
- **C-011 — Provider neutrality.** Домен не зависит от Ollama или облачного vendor. Provider выбирается capability contract и benchmark.
- **C-012 — Local-first and explicit cloud.** По умолчанию пользовательские тексты остаются локально. Внешняя отправка требует выбранного provider и понятного consent.
- **C-013 — No blind retries.** Повторная попытка обязана получать классифицированные findings или иной repair strategy; повтор того же запроса не считается исправлением.

## Перевод и лор

- **C-014 — Quality hierarchy.** Приоритет: техническая целостность → точность смысла → официальный лор/термины → естественный русский → единый стиль.
- **C-015 — Contextual identity.** Строка не идентифицируется только её английским текстом. Unit, source revision и context signature хранятся раздельно.
- **C-016 — Versioned terminology.** Официальный корпус берётся из совместимой локальной установки, имеет provenance и не распространяется с приложением.
- **C-017 — Meaning invariants.** Сущности, числа, знак, отрицание, модальность, причинность и игровые эффекты проверяются отдельно от грамматики.
- **C-018 — Independent review.** Структурная валидность не доказывает качество. Semantic/lore и Russian-language review имеют отдельные findings и тестовые наборы.
- **C-019 — Manual work wins.** Явно одобренный ручной перевод или термин не перезаписывается автоматикой без отдельного пользовательского действия.
- **C-020 — Conservative fallback.** Неразрешённая единица использует предыдущий одобренный перевод либо английский источник с видимым предупреждением; система не выдаёт догадку за готовый русский текст.

## Продукт и инженерная дисциплина

- **C-021 — One source, one companion.** Основной артефакт — отдельный управляемый русификатор на исходный мод. Агрегация является экспортом, не внутренней моделью.
- **C-022 — Incremental by evidence.** Неизменённые source revision и context signature не вызывают модель. Изменение контекста требует повторной проверки даже при том же тексте.
- **C-023 — Persistent jobs.** Pause, cancel, crash и перезапуск не теряют подтверждённую работу и не создают дубли.
- **C-024 — User data stays out of Git.** Моды, переводы, ключи, базы, кэши, логи и generated reports не коммитятся. В репозитории допустимы только маленькие лицензируемые fixtures.
- **C-025 — Regression for every escape.** Дефект не закрыт без минимального воспроизводящего fixture и теста на правильное поведение.
- **C-026 — Layered test evidence.** Golden corpus, round-trip, property, fuzz/mutation, integration и packaged smoke tests дополняют друг друга; один вид теста не заменяет остальные.
- **C-027 — Modular monolith first.** Сетевые сервисы и второй runtime запрещены без измеренного ограничения и ADR.
- **C-028 — Safety kernel before UI/LLM.** Основной интерфейс и массовый перевод не разрабатываются до прохождения M2.
- **C-029 — Provenance is part of output.** Для каждой строки и сборки воспроизводимы source, context, glossary, provider/model, validators и ручные решения.
- **C-030 — No date-driven gate bypass.** Roadmap задаёт зависимости и критерии выхода, а не обещанные даты. Следующий этап начинается только после доказательств текущего.

## Иерархия решений

При конфликте действует порядок:

1. каноны и threat model;
2. принятые ADR;
3. version profile и спецификация формата;
4. критерии milestone;
5. реализация и локальные соглашения.

Старый проект, prompt модели и текущая реализация никогда не являются источником истины сами по себе.
